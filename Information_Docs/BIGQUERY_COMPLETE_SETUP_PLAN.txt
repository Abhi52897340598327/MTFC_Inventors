================================================================================
TROUBLESHOOTING GUIDE
================================================================================

ISSUE 1: "gcloud: command not found"
Solution: 
- Restart PowerShell after SDK installation
- Check PATH: $env:PATH -split ';' | Select-String "Google"
- Reinstall SDK if needed

ISSUE 2: Authentication fails / browser doesn't open
Solution:
- Run: gcloud auth application-default login --no-browser
- Follow instructions to manually copy authorization code

ISSUE 3: "Permission denied" or "Access denied" errors
Solution:
- Verify correct Google account is logged in
- Check project billing is enabled
- Ensure BigQuery API is enabled

ISSUE 4: Query timeout or connection errors
Solution:
- Check internet connection
- Try smaller queries first (LIMIT 1000)
- Increase timeout in script

ISSUE 5: "Project ID not found"
Solution:
- Verify project ID exactly matches (case-sensitive)
- Check: gcloud config list
- Update in script: PROJECT_ID = "exact-project-id"

ISSUE 6: High costs / unexpected charges
Solution:
- Monitor in Console: Billing → Reports
- Should be ~$0.01 total
- Set billing alerts for safety
- Free tier should cover this dataset

ISSUE 7: Data quality issues (missing values, wrong dates)
Solution:
- Check WHERE clause in queries (filters bad data)
- Verify timestamp conversion logic
- Re-download specific cells/PDUs if needed

ISSUE 8: Script hangs or very slow
Solution:
- Normal: 8-12 minutes for full download
- BigQuery processing 115 tables sequentially
- Monitor progress output
- Can interrupt (Ctrl+C) and resume by modifying script
